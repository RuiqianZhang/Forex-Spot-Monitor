# Forex Spot Monitor
  
> [中文说明](README.md) | [English Version](README.en.md)
  
一个强大的 macOS 菜单栏常驻小组件，用于实时查看全球外汇、贵金属、美元指数、原油等金融品种报价。

本项目基于 **Tauri v2 + Vue 3 + TailwindCSS + Rust** 构建，兼具极低系统资源占用与流畅的原生桌面体验。

---

## ✨ 核心特性

- **菜单栏极速概览**：在 macOS 菜单栏实时显示当前选中品种的最新报价与涨跌幅，支持单次点击快速下拉列表切换品种。
- **动态数据驱动机制**：数据源服务器定义与品种列表均由本地外部 JSON 配置驱动，无需修改代码即可灵活扩展源节点。
- **现代化可视化调配中心**：内置采用“毛玻璃”视觉语言的配置窗口。支持可视化增减源节点、定义 HTTP Header / Params / Body 负载，并支持测试连通性。
- **高阶提取引擎**：
  - 支持 **万能 JSON_FIELDS 规则** 提取任意标准的基于 JSON 对象层级返回的报价数据。
  - 支持基于原生底层内存拦截机制进行特定源（如新浪外汇微字符串）的高性能切片。
- **模拟防封策略**：底层网络发起器使用 Rust TLS 封装，支持灵活的浏览器欺骗指纹 (`Impersonate Chrome`) 以及严苛 SSL 豁免与代理。
- **多重人性化交互**：每秒定时自动定滑刷新报价行情、测试数据强制中断打断、中英文界面多语言热切。

---

## 🚀 模块架构

- **后端中枢驱动 (Rust)**：使用 `tauri`, `reqwest`, `curl-cffi` 搭建超轻量网络隧道与底层缓存队列，进行纳秒级内存交互。
- **前端调配展示 (Vue 3)**：结合 `TailwindCSS` 为数据源列表绘制出带有沉浸式流光背景的液态视觉界面（`src/App.vue` 操作中心）。
- **语言国际化 (TypeScript)**：拥有无需重启的跨层动态语言库分发 (`src/i18n.ts`)。

## 🛠️ 安装与开发运行

请确保您的系统已安装了最新的 Node.js (v20+) 以及 Rust 编译环境。

```bash
cd tauri-v2

# 1. 安装前端控制台依赖
npm install

# 2. 启动具备热重载特性的 Tauri 联合调试服务
npm run tauri dev
```

## ⚙️ 数据源与解析器配置指南

本项目的一大核心亮点是无需改代码，通过自带的配置窗口（点击菜单栏中的「设置 -> 编辑配置…」）即可支持市面上绝大多数基于 API (JSON 或纯文本) 的金融数据服务。

### 1. 配置数据源 (Providers)
数据源定义了应用程序**如何向目标服务器发送网络请求**以获取原始数据。

在「数据源配置」标签页中，您可以：
- **定义基础请求**：填写目标的 `URL 模板`（例如 `https://api.example.com/price/{symbol}`，其中的 `{symbol}` 会在请求时自动替换为具体品种代码）、选择 `GET/POST` 方法。
- **自定义 HTTP 载荷**：如果 API 需要鉴权或特殊协议，可展开下方手风琴面板，用标准 JSON 格式填写 `Headers`、`Params`（URL 查询参数）或 `Body`。
- **反爬虫规避**：对于检测严格的源，可启用 `浏览器模拟 (Impersonate Chrome)` 注入类似真实浏览器的底层 TLS 检测握手指纹，或关闭 `SSL 验证` 以绕过不规范的证书节点。
- **直接跑测验证**：在右侧的测试面板中输入一个存在的品种代码并点击「发送请求」，应用会立刻输出该源的原始返回值（`Response Body`）以及网络耗时状态码，帮您确立目标源处于可用状态。

### 2. 配置解析规则 (Parsers & Instruments)
当「数据源」能成功返回数据后，您需要告诉应用**如何从这堆原始返回体中提取您想要的最终涨跌数据**。

切换至「解析规则配置」标签页，在对应的数据源下管理您的品种 (`Instruments`)：
- **建立品种映射**：填写您想要的 `显示名称`（出现在菜单中，如 `黄金`），并在 `品种代号` 中准确填写上游 API 能识别的真实代号（如 `XAU/USD`），分配给您想要的分组（如 `外汇与贵金属`）。
- **选择解析引擎 (Parser Type)**：
  - **内置固定解析引擎** (`sina_forex_text`, `swissquote_bbo` 等)：针对历史主流来源量身打造的底层 Rust 正则直接切割机。如果您的数据格式复合这些老牌平台，直接选用即可，性能极高。
  - **万能 JSON 引擎** (`JSON_FIELDS`)：当您接入非预制的新型或小众 API 时，只要目标返回格式为规范的 JSON，均可选用该引擎。

#### 💡 万能 JSON 引擎 (`JSON_FIELDS`) 路径提取规则
选定 `JSON_FIELDS` 后，您需要明确定义层级路径从而“按图索骥”。路径采用经典的 **小数点分隔** 格式，例如目标接口返回：
```json
{
  "code": 200,
  "data": { "rates": { "last_price": 2730.55, "ask": 2730.60, "bid": 2730.50 } }
}
```
此时您只需进行如下可视化配置：
- **价格路径 (Price Path)** -> 必填，写 `data.rates.last_price`
- **买/卖价 (Bid/Ask Path)** -> 选写，分别写 `data.rates.bid` 和 `data.rates.ask`

**特殊用法：数组与列表项**
对于某些以列表包裹元素的返回值，如 `{"quotes": [{"price": 100}]}`，路径中可直接使用数字下标代表进入数组：
- **价格路径** -> `quotes.0.price`

这就是全新的「配即用」零码接入理念！

## 🔒 配置持久化说明

本地所有由可视化中心修改的数据源模型，均会被持久化加密反序列保存于您的系统级通用应用程序路径下：
- 📂 `~/Library/Application Support/Forex Spot Monitor/data_sources.json`

当发生误删查或损坏时，项目将自动提供降级预设方案的缺省列表接管运作。
